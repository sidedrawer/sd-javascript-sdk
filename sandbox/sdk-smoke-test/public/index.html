SideDrawer SDK Smoke Test

Browser UI for testing @sidedrawer/sdk. Use Mock when running npm run dev (everything is wired to the local mock API). Switch to UAT to hit the real backend with a valid access token.

Configuration

Calls GET /api/v1/records/sidedrawer/sidedrawer-id/{sidedrawerId}/home and reads subscriptionFeatures["sidedrawer.maxUploadMBs"]. Leave empty to let the backend decide (no client-side preflight).

Resumable download (Phase 2)

With Resume from byte > 0 the SDK sends Range: bytes=N-. When the request succeeds you'll see 206 Partial Content and only the remaining bytes come back. With discardBuffer the SDK does not hold the file in memory; chunk count + total bytes are logged but no Blob is produced and no file is saved.
